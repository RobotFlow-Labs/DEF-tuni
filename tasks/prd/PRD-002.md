# PRD-002 — Dataset and Checkpoint Normalization

## Objective
Remove hardcoded dataset assumptions by driving root paths from CLI and environment variables.

## Steps
1. Set dataset root variables:
   - `ANIMA_DATASET_ROOT_FMB`
   - `ANIMA_DATASET_ROOT_PST900`
   - `ANIMA_DATASET_ROOT_CART`
2. Use `def-tuni eval --dataset <DATASET> --checkpoint <path>` for runtime materialization.
3. Confirm generated JSON in `.anima/run_configs/<dataset>/`.

## Done Criteria
1. Runtime config files are generated per dataset.
2. `root` is overridden when env/CLI path is provided.
3. Upstream `repositories/TUNI/configs/*.json` remain unchanged.
